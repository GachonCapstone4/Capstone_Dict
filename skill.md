## 자동화 도구 설계 시 공통 제한사항

1. 각 기능의 도구들을 설계할 시 절대 다른 도구의 항목을 수정/삭제/추가 하지 않는다.

ex) diagnosis에서 network 도메인의 작업을 위해 os의 source에 logic을 추가해서는 안된다.

2. 모든 도구들은 자신의 실행 및 출력 정보를 구조체로 담아 x.sse.fanout에 json 형태로 publish 해야한다.

3. RabbitMQ 리소스는 이미 선언되어 있고 절대로 본 프로세스 안에서 생성하거나 삭제하지 않는다.

4. 같은 프로젝트 안에있지만 k8s job을 통해서 별도의 args 태그로 담당하는 기능만 수행이 가능행해야한다.
